## Theme Selection Methodology

Each candidate theme was scored against **4 criteria (1–10 scale)**:

| Criterion       | Description                                                                |
|-----------------|----------------------------------------------------------------------------|
| Visual Appeal   | Does the palette feel striking and memorable at first glance?              |
| Professionalism | Does it signal developer credibility for a senior-role portfolio?          |
| Modernity       | Is it consistent with 2024–25 premium product design trends?               |
| Pleasantness    | Is it comfortable for extended reading without fatigue?                    |

Eliminated candidates: plain red (#ff0000 base) — appeals 9, professionalism 5 = **rejected**. Generic Material blue — modernity 5 = **rejected**. High-saturation neons — pleasantness 4 = **rejected**. Pure light-grey-on-white — visual appeal 6 = **rejected**.

## Theme Selector Widget — Design Contract

### Placement

- Inside `nav-inner`, right-aligned — between `nav-links` and the mobile `nav-toggle`.
- Uses `flex: 0 0 auto` — does not grow or shrink.
- On mobile (≤768px): visible, placed to the left of `nav-toggle`.

### Trigger Button

- Dimensions: 36×36px
- Shape: `border-radius: var(--radius-pill)` (circular)
- Icon: Inline SVG palette icon, 18×18px
- Background: `rgba(255,255,255,0.06)`
- Border: `1px solid var(--border-glass)`
- Hover: border transitions to `var(--border-hover)`, slight background lift
- Active state (panel open): icon rotates 30deg, border glows with `var(--accent)`
- `aria-label="Change theme"`, `aria-expanded="false|true"`, `aria-haspopup="listbox"`, `id="theme-toggle"`

### Dropdown Panel

- Position: `position: absolute`, `top: calc(100% + 8px)`, right-aligned to trigger
- Dimensions: `width: 192px`, `padding: 8px`
- Style: `background: var(--bg-secondary)`, `border: 1px solid var(--border-glass)`, `border-radius: var(--radius-lg)`, `backdrop-filter: blur(20px)`
- z-index: 1001 (above nav at 1000)
- Open animation: `opacity 0 → 1` + `translateY(-8px) → translateY(0)` — 200ms ease-out
- Close animation: reverse — 150ms ease-in
- Dismiss: click outside, press Escape, click trigger again
- `role="listbox"`, `aria-label="Select theme"`

### Theme Option Rows

Layout per row: `[swatch 20×20px] [label text] [checkmark if active]`

- Row: `padding: 8px 10px`, `border-radius: var(--radius-md)`, `gap: 10px`, `display: flex`, `align-items: center`
- Swatch: `width: 20px; height: 20px; border-radius: 50%; flex-shrink: 0`
- Label: `font-size: 0.875rem; font-weight: 500; color: var(--text-primary); flex: 1`
- Checkmark: SVG 14×14px, `color: var(--accent)`, shown only on active row, `aria-label="Active"`
- Hover: `background: rgba(255,255,255,0.05)`
- Active row: `background: rgba(255,255,255,0.08)`
- `role="option"`, `aria-selected="true|false"`, `data-theme-value="midnight|arctic|forest|dusk"`

### State Persistence

- On select: `localStorage.setItem('portfolio-theme', value)`
- On page load: inline `<script>` in `<head>` (before stylesheet) reads and sets `document.documentElement.setAttribute('data-theme', ...)`
- Default: `"midnight"` if localStorage key absent or invalid

## CSS Architecture

### File: `assets/css/themes.css`

Contains only the 4 `[data-theme]` override blocks. Structure:

```css
:root, [data-theme="midnight"] { /* existing :root vars */ }
[data-theme="arctic"]  { /* Arctic overrides */ }
[data-theme="forest"]  { /* Forest overrides */ }
[data-theme="dusk"]    { /* Dusk overrides */ }
```

Linked in `<head>` after `style.css`.

### File: `assets/js/theme.js`

`ThemeSelector` class: init, buildWidget, togglePanel, selectTheme, persist, handleOutsideClick, handleEscape.

Script linked in `<body>` before `main.js`.

### Transition Smoothness

Apply to `<html>` element during theme switch (then remove after 400ms via JS to avoid sluggish repaints on hover):
```css
html.theme-transitioning * {
  transition: background-color 0.3s ease, border-color 0.3s ease, color 0.2s ease !important;
}
```

## Color — 60/30/10 Rule

Applied per-theme:

| Role          | Allocation | Usage                                                       |
|---------------|------------|-------------------------------------------------------------|
| Dominant 60%  | `--bg-primary` + `--bg-secondary` | Page bg, section bands         |
| Secondary 30% | `--bg-card`, nav bg, panel bg   | Cards, nav, theme panel           |
| Accent 10%    | `--accent`, `--accent-2`, `--accent-3` | Links, CTAs, glows, selector active indicator |

Accent reserved for: nav active link, CTA buttons, chip borders, hero glow rings, stat-number gradients, checkmark in selector, trigger border on-active.

Destructive: not applicable in this phase.
