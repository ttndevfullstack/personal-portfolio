# Plan 05-01: Implement Multi-Theme System & Selector UI

## Objective

Implement all 4 CSS variable themes (Midnight, Arctic, Forest, Dusk) and a glassmorphic theme selector widget in the navigation bar. Themes are applied via `data-theme` attribute on `<html>`, persisted to `localStorage`, with an inline `<head>` script to prevent FOUC.

## Tasks

### Task 1 — Create `assets/css/themes.css`
Create a new CSS file with 4 `[data-theme]` blocks containing all token overrides from the UI-SPEC. The existing `:root` in `style.css` serves as the Midnight default — only the 3 override themes need their full token sets.

### Task 2 — Create `assets/js/theme.js`
Implement the `ThemeSelector` class: builds the widget DOM, handles open/close, theme selection, localStorage persistence, keyboard navigation (Escape, ArrowUp/ArrowDown), and outside-click dismissal.

### Task 3 — Update `index.html`
1. Add inline FOUC-prevention script in `<head>` (reads localStorage, sets `data-theme` immediately).
2. Link `assets/css/themes.css` in `<head>` after `style.css`.
3. Link `assets/js/theme.js` in `<body>` before `main.js`.
4. Add `id="theme-btn-container"` placeholder `<div>` inside `.nav-inner` for the widget mount point.

## Verification

- All 4 themes produce visually correct color changes when selected.
- Selected theme survives page refresh.
- No FOUC on load (check with DevTools → throttle network).
- Keyboard: Tab → trigger, Escape → closes panel, ArrowUp/ArrowDown navigate options.
- Mobile (≤768px): selector is visible and functional alongside hamburger menu.
